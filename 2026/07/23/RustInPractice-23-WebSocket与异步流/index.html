RustInPractice（23）：WebSocket 与异步流

发表于2026-07-23|更新于2026-07-20|RustInPractice
|浏览量:

WebSocket 在 HTTP upgrade 后保持双向消息通道。tokio-tungstenite 将连接同时实现 Stream 与 Sink，接收用 next().await，发送用 send().await。

对应 networking/websocket。下一篇：SQLite。

服务端通常为每条连接 spawn 一个 task，连接结束后 task 自动结束。需要显式处理 text、binary、ping、pong、close 和错误；demo 将 binary 转成文本提示，因此它是文本 echo，不是保留二进制 payload 的 echo 服务。

accept 循环也不应静默吞掉 listener 错误。生产服务还需要连接上限、超时、认证和关闭协议。

下一篇：SQLite 与 rusqlite。
